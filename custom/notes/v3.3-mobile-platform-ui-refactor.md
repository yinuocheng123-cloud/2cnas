# 版本记录：v3.3-mobile-platform-ui-refactor

## 1. 本轮目标

本轮目标是按用户提供的移动端 UI 参考气质，优化 CNAS认可指南成交页的手机端首页体验。

本轮不是新增业务功能，也不是把页面改成强营销落地页，而是把手机端做成更接近“CNAS行业服务平台”的高级移动端入口：深蓝高级风、克制、专业、有秩序、大留白、大标题、卡片化，并用金色作为少量点缀。

桌面端保持现有结构和风格，不进行大改。

## 2. 背景与上下文

本轮承接 `v3.2-layout-hardening.md`：

- v3.2 已完成低分辨率、Windows 缩放、旧桌面设备和窄宽度环境下的布局加固。
- 断点已明确为 `<=768px` 手机、`769~1024px` 平板、`>1024px` 桌面。
- 已将多处固定列 grid 改为自适应 grid。
- 本轮必须保留 v3.2 的布局稳定性成果，不重新引入横向溢出、卡片压缩和移动端误触发问题。

本轮同时延续 v3.1 的业务结构：

- 企业微信优先承接。
- 在线表单兜底。
- A/B/C 初步诊断。
- UTM 采集。
- CNAS认可后维护模块。
- FAQ。

## 3. 问题分析

修改前手机端主要问题是：页面虽然已经响应式可用，但本质仍是桌面页面的缩小版，移动端没有形成清晰的“行业服务平台入口”节奏。

具体表现：

- 首屏偏服务介绍和转化表达，平台定位不够突出。
- 行业问题、平台能力、问卷和内容入口没有按移动端阅读顺序重新组织。
- 手机端信息密度偏高，模块连续堆叠后容易像传统咨询落地页。
- 企业微信二维码虽然存在，但需要在手机端更克制地作为顾问入口，而不是强销售入口。

核心矛盾是：需要在不破坏桌面端和业务逻辑的前提下，让手机端拥有独立的视觉秩序和平台感。

## 4. 候选方案比较

### 方案一：直接替换全站首页结构

核心思路：按参考图彻底重做 HTML 顺序和模块。

优点：移动端可控性最强。

缺点：会明显影响桌面端，也可能破坏 v3.2 的布局稳定性，不符合“只优化移动端”的要求。

结论：放弃。

### 方案二：只改 CSS，不新增移动端内容块

核心思路：保留原 DOM，只通过 CSS 深蓝化、卡片化。

优点：改动范围小。

缺点：无法补出用户要求的移动首屏平台定位、行业问题 6 卡、平台能力 4 卡和最新行业内容 3 条入口。

结论：不足以满足本轮目标。

### 方案三：新增 mobile-only 结构并用 CSS 控制展示

核心思路：新增少量移动端专属内容块，桌面端隐藏；手机端隐藏部分重复桌面长模块，并用 `order` 调整移动端阅读顺序。

优点：移动端可获得独立体验，桌面端结构基本不变；业务逻辑和表单 JS 不受影响。

缺点：HTML 中会存在少量移动端专属内容，与桌面内容形成双轨维护。

结论：采用。

## 5. 最终决策

本轮采用 mobile-only 结构 + CSS 媒体查询控制：

- 桌面端继续使用原 hero、三大核心系统、知识库、流程和 footer。
- 手机端新增平台化首屏、行业问题卡片、平台能力卡片和最新行业内容入口。
- 手机端隐藏重复或过重的桌面模块：指标背书、场景模块、服务流程、底部强 CTA、handoff 说明。
- 手机端保留并优化企业微信顾问入口、路径判断问卷、诊断结果、FAQ、维护模块和 footer。

## 6. 具体实现

涉及文件：

- `index.html`
- `styles.css`
- `custom/notes/v3.3-mobile-platform-ui-refactor.md`

具体改动：

- 在首屏新增 `mobile-platform-hero`，仅手机端显示，包含平台定位、核心观点、四个能力标签和克制主入口。
- 在风险模块新增 `mobile-issues`，仅手机端显示 6 个行业问题卡片。
- 在核心系统模块新增 `mobile-platform-capabilities`，仅手机端显示 4 个平台能力卡片。
- 在知识库模块新增 `mobile-latest-content`，仅手机端显示 3 条行业内容入口，链接均指向已有锚点，不编造真实文章页面。
- 在路径判断区域新增 `mobile-assessment-intro`，手机端以“CNAS认可路径判断问卷”为主标题。
- FAQ 去掉默认展开状态，使移动端更接近折叠式 FAQ。
- footer 增加移动端底部链接与 copyright 展示，并保持备案号 `浙ICP备2020044218号-3`。
- CSS 中新增移动端深蓝平台视觉、金色点缀、卡片圆角、模块排序和移动端隐藏规则。

未改动：

- `script.js` 未修改。
- `LEAD_FORM_ENDPOINT` 未修改。
- 表单字段、payload、UTM、A/B/C 诊断逻辑未修改。
- 企业微信二维码路径未修改。
- 桌面端主体结构未重排。

## 7. 本轮优点

- 手机端从“桌面缩小版”变为更清晰的平台入口。
- 首屏不再堆表单和大二维码，更符合“CNAS行业服务平台”的克制定位。
- 行业问题、平台能力、顾问入口、问卷、内容、FAQ、维护形成更适合手机阅读的顺序。
- 保留 v3.2 的断点与溢出控制成果。
- 没有新增依赖，没有改动业务 JS，表单和诊断逻辑风险低。

## 8. 本轮缺点与代价

- mobile-only 内容块会带来少量双轨维护成本，后续如果改平台定位文案，需要同时注意桌面与移动端。
- 本轮为了保护桌面端，没有把全部桌面模块重做为统一设计语言，因此移动端和桌面端视觉差异会比过去更明显。
- 最新行业内容目前使用已有锚点作为入口，不是真实文章详情页；后续如有文章系统，需要替换为真实链接。
- 本轮没有新增真实电话、邮箱、微信或接口配置。
