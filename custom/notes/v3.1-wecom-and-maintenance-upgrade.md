# 版本记录：v3.1-wecom-and-maintenance-upgrade

## 1. 本次目标

本次目标是在现有 CNAS认可路径判断成交页基础上完成两类轻量升级：

1. 将成交页升级为“企业微信优先承接 + 在线表单兜底 + 增长中台可接入”的线索承接结构。
2. 在不改变“先判断认可路径”主线的前提下，补充“CNAS认可后维护”内容，让页面具备首次认可与后期维护的全周期表达。

本轮不新增复杂依赖，不接入真实接口，不编造真实二维码、电话、域名或案例。

## 2. 实际改动内容

- 调整主 CTA 为“添加顾问，领取CNAS认可路径判断问卷”。
- 精确更新企业微信承接卡片，包含指定标题、副标题、说明、二维码占位图和“我已添加，继续填写基础信息”按钮。
- 保留在线表单作为兜底入口，并将表单标题改为“先在线填写基础信息”。
- 调整新增字段选项：认可范围是否明确、人员设备是否基本具备、是否已添加企业微信。
- 重写 A/B/C 诊断等级判断逻辑，使其符合当前阶段和启动时间优先级。
- 在页面中后段新增“CNAS认可后维护”模块，补充体系运行维护、监督评审与复评审准备、扩项与变更支持、年度合规维护包四类内容。
- FAQ 新增 CNAS认可通过后是否需要维护、认可后维护适合哪些实验室两个问题。
- 更新增长中台文档，补充企微导流、企微发问卷、诊断结果、中台字段、标签体系、跟进 SOP 和企微 API 接入建议。
- README 补充认可后维护模块用途说明。
