# 增长中台接入设计：CNAS路径判断成交页

## 1. 目标

本文件说明当前静态成交页如何与后续增长中台衔接。当前页面优先引导用户添加企业微信顾问，在线表单作为兜底入口；暂不接入真实数据库，也不编造真实接口。

## 2. 当前线索路径

1. 用户进入页面后，主 CTA 引导添加企业微信顾问。
2. 用户也可以填写在线路径判断问卷。
3. 表单提交后，前端生成 A/B/C 初步诊断等级。
4. payload 保留来源页面、UTM 参数、提交时间、表单字段和初步诊断结果摘要。
5. 未配置真实 `LEAD_FORM_ENDPOINT` 时，页面执行模拟提交并在控制台输出 payload。

## 3. 网站如何导流企微

页面首屏和诊断结果底部都使用“添加顾问”类轻转化 CTA，将高意向用户优先导向企业微信顾问。企业微信卡片显示二维码占位图，正式投放前应替换为真实企业微信二维码或企业微信活码。

在线表单保留为兜底入口，用于用户暂时不方便添加企业微信时提交基础信息。

## 4. 企微如何发问卷

建议顾问在用户添加企业微信后，发送以下内容：

1. CNAS认可路径判断问卷链接。
2. 说明先判断认可范围、人员设备和启动阶段，再决定是否进入材料或申请动作。
3. 提醒用户补充当前阶段、认可范围、资源准备状态和计划启动时间。

## 5. 问卷如何生成诊断结果

前端根据当前阶段、计划启动时间、认可范围清晰度和人员设备准备度生成 A/B/C 等级：

- A类：适合进入认可路径设计阶段。
- B类：适合先准备，不建议马上申请。
- C类：暂不适合直接启动申请。

诊断结果同时展示当前判断、主要风险、建议优先动作和下一步建议。

## 6. 线索如何进入增长中台

当 `LEAD_FORM_ENDPOINT` 配置为真实接口后，页面会通过 `fetch` 提交 JSON payload。接口可以接入飞书表格、金数据、企业自有增长中台或轻量后端 API。

未配置真实接口时，页面只模拟提交并在浏览器控制台输出 payload，不保存真实线索。

## 7. 建议中台字段

- company：企业名称
- contact：联系人
- phone：手机号
- labType：实验室类型
- stage：当前阶段
- scopeClarity：认可范围是否明确
- resourceReadiness：人员设备是否基本具备
- wecomAdded：是否已添加企业微信
- concerns：最担心的问题
- timeline：计划启动时间
- notes：补充说明
- source：来源标识
- sourcePage：来源页面
- utm：UTM 参数
- submittedAt：提交时间
- diagnosisSummary：初步诊断摘要与 A/B/C 等级

## 8. 标签体系

建议进入增长中台后按以下标签分层：

- 线索来源：自然访问、搜索、广告、转介绍、私域。
- 当前阶段：刚开始了解、已准备建设、已做体系文件、准备申请、评审后整改。
- 诊断等级：A类、B类、C类。
- 企业微信状态：已添加、未添加。
- 服务方向：首次认可、评审准备、整改闭环、监督评审、复评审、扩项维护。

## 9. 跟进 SOP

1. A类线索：优先联系，确认申请时间、认可范围和评审准备缺口。
2. B类线索：发送路径判断问卷和准备清单，协助收拢范围与资源差距。
3. C类线索：先做教育和基础判断，不直接推动申请动作。
4. 已添加企业微信线索：优先通过企业微信承接，补充资料后再进入表单或中台。
5. 未添加企业微信线索：短信或电话提醒添加顾问，避免后续沟通断点。

## 10. 后续企微 API 接入建议

后续可接入企业微信活码、企微客户联系 API、飞书表格、金数据或自有增长中台 API。建议先保持静态页稳定，再逐步接入：

1. 企业微信活码：替换 `WECOM_QR_IMAGE`。
2. 客户联系 API：用于记录添加状态和跟进人。
3. 表单 API：替换 `LEAD_FORM_ENDPOINT`。
4. 中台标签：根据 payload 自动打标签并分配跟进优先级。

## 11. 正式接入前配置

需要替换：

- `WECOM_QR_IMAGE`：真实企业微信二维码图片路径。
- `WECOM_CONTACT_TEXT`：真实企业微信承接文案。
- `LEAD_FORM_ENDPOINT`：真实表单提交接口。

未配置真实接口前，页面只能模拟提交，不会保存线索。

## 12. 后续建议

1. 优先完成真实企业微信二维码和表单接口配置。
2. 在投放链接中统一使用 `utm_source`、`utm_medium`、`utm_campaign`、`utm_content`、`utm_term`。
3. 在增长中台按 A/B/C 等级设置跟进优先级。
4. 上线后使用真实表单提交测试验证 payload 字段完整性。
